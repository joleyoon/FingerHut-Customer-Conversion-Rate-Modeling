# Supporting Notebooks

These files preserve the exploratory work behind the group deliverables.
They are grouped by contributor so the core workflow in `src/` remains easy to
navigate.

- `charlie/`: early data-understanding work.
- `isaiah/`: exploratory analysis, modeling, and forecasting work in R.
- `joel/`: Python notebooks for course milestones and forecasting.

Some notebooks reference private competition data or generated outputs that are
not committed. They are retained as development records rather than polished
entrypoints. Start with `src/` for the curated modeling workflow and
`reports/project-summary.md` for the group narrative.
